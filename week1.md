# J224 이화정

## 척추요청 프로젝트

**배경**

미션에 집중하느라 올바르지 않은 자세로 작업을 지속하게 되어 허리, 어깨, 목에 뭉침이나 통증이 자주 발생한다. 이를 개선하기 위해 AI의 도움을 받아 꾸준한 스트레칭 및 가벼운 운동 습관을 들이고자 한다.

**목적**

매일 10~20분씩 스트레칭이나 가벼운 운동을 꾸준히 실천하는 습관을 만든다.

**달성 기준**

- 매일 1회, AI에게 현재 아픈 부위나 컨디션을 전달하고 10~20분짜리 운동 또는 스트레칭 유튜브 영상 1개를 추천 받는다.
- 추천받은 영상을 보며 운동 또는 스트레칭을 실천한다.
- 실천 후, 간단한 후기를 남긴다. (슬랙이나 팀원들과 공유, 아니면 회고 같은 곳에 남기기)

**프롬프트 예시**

- "현재 목과 어깨가 뻐근해요. 15분 정도 할 수 있는 스트레칭 유튜브 영상 추천해줘."
- "허리가 자주 아파서 가벼운 운동을 하고 싶은데, 10분 내외 운동 영상 알려줄래?"
- "오랜 시간 앉아있어서 뭉친 근육을 풀어줄 스트레칭 영상 20분짜리 하나 추천해줘."

## Ai 커밋 메시지 추천 프로젝트

**배경**

효과적인 협업과 코드 관리에 있어서 커밋 메시지는 매우 중요하지만, 일관된 커밋 컨벤션을 지키며 메시지를 작성하는 것은 생각보다 쉽지 않다. 특히 초보 개발자나 협업 경험이 적은 팀원들은 어떻게 메시지를 작성해야 할지 고민하는 경우가 많다. 이에 AI의 도움을 받아 커밋 컨벤션에 맞는 메시지를 추천받아 생산성을 높이고 커밋 품질을 향상시키고자 한다.

**목적**

- 커밋 컨벤션에 부합하는 커밋 메시지를 꾸준히 추천받아 적용하며 작성 습관을 개선한다.
- AI를 활용해 커밋 메시지 작성 부담을 줄이고 협업 시 커밋 로그 가독성을 높인다.

**달성 기준**

- AI에게 현재 작업 중인 기능이나 변경사항을 간단히 전달하고 커밋 컨벤션에 맞는 메시지 3개 이상을 추천받는다.
- 추천받은 메시지 중 적합한 메시지를 선택하여 미션 해결 과정에서 사용한다.

**프롬프트 예시**

- "이번에 로그인 기능에 사용자 인증 로직을 추가했어. 커밋 컨벤션에 맞는 메시지 3개 추천해줘."
- "버그 수정으로 네트워크 요청 에러 핸들링 코드를 고쳤어. 깔끔한 커밋 메시지 3개 부탁해."
- "UI 버튼 스타일을 변경하고 레이아웃을 개선했어. 컨벤션에 맞는 커밋 메시지를 3가지 알려줘."

## 릴레이 프로젝트 토론에서 공유할 주제

### 재미있는 사례

나는 홈어시스턴스에 LLM을 연결해 스마트홈 자동화에 사용하고 있는데, 이걸로 뭔가 할 수도 있지 않을까?

### 가깝고도 먼 분야에서 일어나는 일들

[leesung studio](https://www.instagram.com/leesungstudio/): 예술가 소개. 공연 및 설치 예술을 하는 분이다. 우리가 하는 개발과는 다른 접근이라 흥미롭다. 관객 앞에서 3명이 라이브 코딩 공연을 한다. Google Docs 처럼 여러 개의 커서로 3명이 동시에 코드를 편집하는 화면을 중간에 둔다. 양 옆에는 LLM과의 대화창을 띄워둔다. "나는 지금 관객 앞에서 코드로 사운드를 디자인하고 있어. 너와 나눈 대화를 바탕으로 사운드를 변화시킬거야"같은 프롬프트로 상호 작용한다. 결국 LLM, 예술가, 관객이 한 공간에서 상호 작용한다. 예술 분야에서는 AI로 생성한 이미지 때문에 기술에 대한 인식이 안 좋고, 영화 산업에서도 파업이 이어졌다. LLM은 그 가능성이 무궁무진한 흥미로운 기술인데, 이를 둘러싼 자극적인 사건들이 펼쳐지고, 또 부적절한 방식으로도 악용되며 사회적인 인식에 영향을 준다. 나는 이런 흐름이 안타깝기도 하고, 또 충분히 공감되기도 한다.

AI 연구 기관에서 웹 서버를 긁어 자료를 수집할 때 `robot` 파일을 무시하고 서버에 부담을 주는 방식으로 크롤러를 보내기도 한다. 결국 이런 무분별한 집단에 보복하는 것도 가능하다는 연구도 나왔다. 학습 데이터의 품질을 떨어뜨려 poisoning하는 데이터로 답변하는 방식이다. 이렇게 웹 개발자와 AI 개발자 사이의 미묘한 갈등도 있다. 코딩 테스트 플랫폼의 게시판에서도 AI 연구자들이 채점 서버에 부담을 지우는 것에 대한 원성이 있다. 우리는 이 기술 발전을 둘러싼 사회적 환경과 맥락을 어떻게 바라봐야 할까? 개발자니까 기술적인 측면에만 집중해 '새로운 가능성'에 집중하는 것이 중요할까? 사회에서 신기술이 모두에게 이로운 방향으로 활용될 수 있도록 고민하며 새로운 책임을 받아들여야 하지 않을까?

### 학습에 활용

나는 학습 방향을 정할 때 AI를 활용한다. Rabbithole를 탐색할 때 어떻게 도움이 되었는지, LLM 등장 전/후 학습 방식이 어떻게 바뀌었는지 토론해보자.

## 퀘스트 배경

나는 옵시디언 플러그인으로 LLM 출력을 읽기 쉽게 다듬는 정규식을 만들어 사용중이다. 이모지, em-dash `—`, horizontal rules `---`, bold, 줄바꿈 등 AI 티가 나는 부분을 제거하고, 읽기 쉬운 포멧으로 바꿔 기록을 남긴다. 전부 자동화를 했는데, 딱 하나 못한 것이 있다. **나를 매우 칭찬하는 문장들을 찾아 지우는 건 직접 읽어보며 제거하고 요점만 남게 다듬어야 한다.**

요즘 인터넷 댓글을 보면 LLM이 칭찬하는 "탁월한 생각이에요, 바로 그거에요"같이 **과장된 표현을 꼬집는 농담**도 보인다. 그런데, 지난 한 주 피어 피드백을 하며 **동료분들한테 좋은 반응을 받았을 때는 정말 기분이 좋았다.**

이게 매우 신기한 지점 같다. **나는 어쩌면 AI를 동료로 인정하지 않는지도 모른다.** '통계 모델 주제에 사람처럼 대해주길 바라는거야?' 또는 '넌 늘 거짓말만 하니 일단 안 믿고 보겠어' 라는 자세를 취하고 있는 것 같다.

우리가 AI한테 듣기 싫어하는 "**칭찬 표현**"들은 어쩌면 부스트캠프 커뮤니티에 긍정적인 영향을 주는 재료가 될 수 있을 것 같다. 우리 모두 미션을 하며 서로가 얼마나 최선을 다해 고민하고 도전하고 있는지 알기에 서로를 더 인정하게 된다. 그런 동료에게 받은 칭찬은 껍데기만 있는 빈 칭찬과는 다르게 느껴진다.

**칭찬을 AI로 생성하자는 말이 아니다. AI가 칭찬하는 방식에서 보고 배울 점을 찾아보자는 뜻이다.** AI는 부족한 나를 보고도 어떻게든 칭찬할 거리를 찾아낸다. 그걸 래퍼런스로 삼아 어떻게 동료에게서 장점을 발견할 수 있을지 배울 수 있을 것 같다. 물론 우리는 사람이니까 서로의 감정을 헤아려 보다 **진정성**이 느껴지도록 접근해야 될 것 같다.

## 어떻게 퀘스트로 만들 수 있을까?

### 아이디어 1

AI의 칭찬 방식을 직접 경험해보며 분석하고, 어떤 점이 고래를 춤추게 만드는지 고민해보자. 그걸 나름대로의 기준으로 정리해 커뮤니티 활동의 체크리스트처럼 활용해도 좋을 것 같다.

**예시**

```
동료 피드백 체크리스트
 - 동료가 가장 열심히 한 부분을 발견하고 언급했다.
 - 무조건 모든 것에 동의하기보다 잘한 점을 먼저 칭찬하고, 의견이 다른 부분은 나중에 덧붙인다.
 - 일회성으로 던지는 칭찬이 아니라, 정말 오랜 시간을 들여 자세히 바라보았다는 느낌이 들 수 있게 디테일한 부분을 언급한다.
 - ...
```

### 아이디어 2

재미있는 게임처럼 진행해도 될 것 같다. AI로 생성한 칭찬과 사람이 생성한 칭찬을 섞어놓고 어떻게 사람인지 맞춰보는 활동.

### 아이디어 3

AI가 작성한 피드백과 내가 직접 작성한 피드백 두개를 동료에게 전달한다. 그 중 더 마음에 드는 피드백을 골라 달라고 부탁한다. '내가 한 피드백보다 AI가 한 피드백이 더 좋다고?'라며 더 고민하는 기회가 될 것 같다. (좋은 칭찬이 뭔지, 껍데기만 있는 칭찬이 뭔지 배울 수 있다)

## 좋은 피드백 남기기

매일 동료 1명에게 '둘 중 더 나은 피드백을 선택해 달라'고 요청해 보세요. 이 퀘스트에 대해 간략히 소개를 덧붙여도 좋습니다.

```
피드백 1: 내가 직접 동료에 대해 직접 작성한 피드백

```

둘 중 어느 것이 사람이 작성한 것인지 쉽게 맞출 수 없도록 AI 티가 나는 표현은 가능한 제거하고 비슷한 형식으로 전달합니다[.](https://www.instagram.com/reel/DMO0TOkRTrA/?igsh=Zms0OWhzOGw2ZG10) 이 퀘스트의 목적은 AI가 우리를 어떤 방식으로 "칭찬"하는지 직접 경험해보며 동료 피드백 능력을 기르는 것 입니다.
